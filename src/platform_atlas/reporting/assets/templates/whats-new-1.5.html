Platform Atlas

What's New

Version 1.5 · April 2026
Sessions

Sessions tie everything together

When you create a session, you now select an environment, ruleset, and profile in one step. These are bound to the session — when you switch sessions later, the entire context switches with it.

No more forgetting to switch the ruleset after switching the environment. One command, full context restored.

session create prod-audit-q1 session switch

Environments

Organization name per environment

Each environment now carries its own organization name. When you audit Acme Corp's production system and then switch to Beta Inc's staging, the reports automatically use the correct organization — no manual editing.

The global organization name in config.json is used as a default when creating new environments.

env edit to add an org name to existing environments.

Dashboard

Redesigned dashboard

The dashboard now shows your active session's full context at a glance — organization, environment, ruleset, profile, pipeline progress, and your next step. No more guessing which environment you're pointed at.

The sessions table shows environment, organization, and profile columns so you can tell your sessions apart without opening each one.

Data Collection

Gateway4 direct API collection

Atlas now connects directly to the Gateway4 REST API for runtime configuration data. This is faster and more reliable than parsing config files over SSH.

SSH config file collection is still there as an automatic fallback. If the API isn't reachable, Atlas seamlessly tries SSH — no manual setup or flags needed.

📝

Better report exports

JSON and Markdown exports now include the environment name in metadata. Organization name flows correctly through the full pipeline.

⌨️

Keyboard navigation

Arrow keys and Tab now navigate between rule entries in the report's detail modal window. Severity tooltips explain rule impact.

📖

Knowledge Base by default

Remediation steps from the Knowledge Base now appear by default in report detail modals. Suppress with --no-fixes if needed.

Upgrading from an older version?

Existing sessions and environments continue to work without any changes. Sessions created before v1.5 won't have bound metadata, but you can backfill it with session repair.

To add an organization name to your existing environments, use env edit.
